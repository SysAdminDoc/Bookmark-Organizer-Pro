# v6.6.10 Updater Check Audit

Date: 2026-06-06

## Research Findings

- Installed `tufup>=0.10,<0.11` resolved to tufup 0.10.0.
- The installed package metadata shows tufup 0.10.0 depends on `tuf==4.0.*`;
  BOP should keep the optional dependency on tufup and avoid a direct tuf pin.
- `tufup.client.Client.check_for_updates()` refreshes trusted metadata and
  returns target metadata without downloading or applying updates.
- `tufup.client.Client.download_and_apply_update()` is the download/apply path
  and remains intentionally unwired.

## Changes Shipped

- Added `UpdateCheckResult` for structured check output.
- Added updater cache path helpers for metadata, targets, and trusted
  `root.json`.
- Updated readiness checks to require local trusted root metadata.
- Added `UpdateManager.check_for_updates()` with injectable client class for
  testability.
- Updated `updates check` to report available, no-update, not-ready, and
  check-failed outcomes.
- Bumped release metadata to v6.6.10.
