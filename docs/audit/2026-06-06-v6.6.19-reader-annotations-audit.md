# v6.6.19 Reader Annotation Foundation Audit

Date: 2026-06-06
Scope: R-21 partial, reader highlight storage and export foundation

## Objective

Start R-21 with a durable annotation layer over existing extracted text so the
future desktop reader pane has a tested storage and export contract.

## Changes

- Added `reader_annotations.json` as the persisted reader highlight store.
- Added `ReaderHighlight` records with bookmark ID, character range, selected
  text, color, note, and timestamps.
- Added four supported highlight colors: yellow, green, blue, and pink.
- Added range validation against bookmark extracted text before a highlight is
  stored.
- Added per-bookmark Markdown export for highlight quotes, ranges, colors, and
  notes.
- Added the `reader` CLI command group: `list`, `add`, `note`, `delete`, and
  `export`.

## Compatibility

- Existing bookmark records are unchanged.
- Reader annotations live in a separate JSON file, so the feature degrades
  cleanly for bookmarks with no extracted text.
- Markdown export writes to the normal exports area unless a CLI output
  directory is supplied.

## Remaining R-21 Work

- Add a desktop reader pane for extracted text.
- Add in-text selection UI for creating highlights.
- Add interactive note editing and deletion in the GUI.
- Add Markdown export access from the GUI.
