# v6.6.20 Reader Pane Audit

Date: 2026-06-06
Scope: R-21 completion, desktop reader pane and highlight editing

## Objective

Finish R-21 by adding a desktop reader pane on top of the reader annotation
store shipped in v6.6.19.

## Changes

- Added a Tk-native reader dialog for selected bookmarks.
- Rendered extracted text in a read-only `tk.Text` widget with saved highlight
  tags.
- Added selection-based highlight creation with yellow, green, blue, and pink
  colors.
- Added note viewing/editing, highlight deletion, and saved highlight list.
- Added Markdown export from the reader dialog.
- Added reader entry points to the Tools menu, bookmark context menu, and
  command palette.

## Compatibility

- Bookmarks without extracted text still open the dialog but cannot create a
  valid highlight until text exists.
- Reader annotations continue to live outside the bookmark JSON payload.
- The CLI reader workflow from v6.6.19 remains unchanged.

## Follow-Up Candidates

- Add keyboard shortcuts inside the reader dialog after UX testing.
- Add theme-aware title-bar polish from the under-consideration list.
- Add optional import/export of reader annotations with Obsidian export.
