# v6.6.26 Updater Staged Cleanup Audit

## Scope

R-41 advanced from apply preflight to safe cleanup of staged updater artifacts.
The cleanup path removes staged state without touching trusted metadata or
application files.

## Changes

- Added `UpdateCleanupResult`.
- Added `UpdateManager.clear_staged_update()`.
- Added `updates clean-staged`.
- Scoped cleanup to `updates/staged_update.json` and staged target files
  reported by the manifest.
- Documented staged cleanup in the updater bootstrap guide.

## Safety Boundaries

- Cleanup uses the existing target-cache boundary guard before deleting staged
  target files.
- No metadata in `updates/metadata/` is removed.
- No extraction is performed.
- No installer is launched.
- No executable is run.
- No application files are replaced.
- `updates apply` remains blocked.
