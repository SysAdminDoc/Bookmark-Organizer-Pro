# v6.6.15 Provider Streaming Audit

Date: 2026-06-06
Scope: R-15 partial, provider-native streaming for RAG chat

## Objective

Move `chat_with_collection_stream` beyond completed-answer chunking by adding
a provider streaming interface below the RAG service. This keeps existing
callers on `complete()` while giving streaming-aware paths a native delta
source when the selected provider supports it.

## Changes

- Added `AIClient.stream_complete()` with a default fallback that yields the
  existing completed answer.
- Added native streaming adapters for OpenAI-compatible chat providers:
  OpenAI, Groq, and DeepSeek.
- Added native line-delimited streaming for local Ollama `/api/generate`.
- Added `ChatStreamResult` so RAG callers can tell whether events came from a
  native provider stream or from completed-answer chunking.
- Refactored `CollectionChat` prompt preparation, citation sanitization, cache
  handling, and turn finalization so `ask()` and `stream_answer()` share the
  same retrieval path.
- Updated `chat_with_collection_stream` to return `mode=provider_stream_events`
  and `provider_streaming=true` when the RAG service used native provider
  deltas.

## Compatibility

- Existing `complete()` calls are unchanged.
- Providers without a native streaming adapter still work through the fallback
  `stream_complete()` implementation.
- `chat_with_collection_stream` continues to include the full `answer` string
  alongside ordered events for clients that need a non-streaming fallback.
- If citation sanitization changes the joined provider text, RAG falls back to
  sanitized answer chunks so event text still reconstructs the returned answer.

## Remaining R-15 Work

- Add FastMCP progress notifications or transport-level streaming around the
  provider delta stream.
- Decide whether to add native streaming adapters for Anthropic and Google
  after verifying the installed optional SDK surfaces.
- Add a client-side MCP smoke once a local MCP client path can assert progress
  notifications.
