# v6.6.23 Updater Download Audit

## Scope

R-41 advanced from metadata-only checks to trusted target staging. The app can
now download selected tufup targets into the local update cache after the same
readiness gates used by `updates check`.

## Changes

- Added `UpdateDownloadResult` and `UpdateManager.download_update()`.
- Reused the existing disabled-by-default policy, HTTPS repository validation,
  optional tufup dependency check, and trusted `root.json` readiness gate.
- Called `Client.download_target()` only after `check_for_updates()` returns a
  target and tufup exposes selected target metadata.
- Validated reported staged paths against the local `updates/targets/` cache.
- Changed `updates download` from a refusal message to a staging command that
  reports not-ready, no-update, failed, or staged states.
- Kept `updates apply` blocked.

## Safety Boundaries

- No extraction is performed.
- No installer is launched.
- No app files are replaced.
- No updater apply path is invoked.
- Staged target paths must remain inside the target cache before being reported.
