# v6.6.1 MCP Dependency Audit

Date: 2026-06-06

## Research Findings

- FastMCP 3.4.2 is the current PyPI release as of 2026-06-06.
- FastMCP 3.4.x requires Python 3.10+, matching BOP's runtime floor.
- Installing FastMCP 3.4.2 pulled MCP SDK 1.27.2, so BOP's optional MCP floor should be at least MCP 1.24 rather than the previous MCP 1.0 floor.
- MCP draft stateless work remains active and larger than a dependency bump; R-58 should stay open for a protocol-focused pass.

## Code Findings

- `pyproject.toml` allowed `fastmcp>=2.0`, which could install an older FastMCP line despite the roadmap requiring 3.4+.
- `requirements.txt` installed `mcp` but not `fastmcp`, so a "full feature set" install could still fall back to the raw SDK path.
- PyInstaller hidden imports included `mcp` but not `fastmcp`.
- `_build_fastmcp_server()` already works with FastMCP 3.4.2, so the code surface only needed dependency and packaging alignment.

## Changes Shipped

- Bumped optional MCP dependencies to `fastmcp>=3.4,<4` and `mcp>=1.24,<2`.
- Added `fastmcp` to the recommended requirements file.
- Added `fastmcp` to PyInstaller hidden imports.
- Bumped release metadata to v6.6.1.
- Added tests for declared MCP dependency floors and FastMCP builder availability.

## Verification
