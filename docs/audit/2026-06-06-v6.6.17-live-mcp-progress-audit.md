# v6.6.17 Live MCP Progress Audit

Date: 2026-06-06
Scope: R-15 partial, live FastMCP progress bridge and client smoke

## Objective

Move FastMCP chat progress from post-result replay toward live chunk progress.
The chat stream tool now forwards chunk callbacks while the tool is running and
has an in-process FastMCP client smoke to verify progress handler delivery.

## Changes

- Added an optional chunk callback to `CollectionChat.stream_answer()`.
- Added `emit_chunk_events()` for fallback and cached response event callbacks.
- Added `t_chat_stream(..., on_event=...)` so MCP wrappers can observe chunk
  events without changing the public tool schema.
- Added `_call_chat_stream_with_progress()` to run the chat stream tool in a
  worker thread while the async FastMCP wrapper forwards chunk events through
  `ctx.report_progress()`.
- Updated the FastMCP `chat_with_collection_stream` wrapper to use the live
  progress bridge.
- Added an in-process FastMCP `Client(app)` smoke that verifies progress handler
  calls and the final `provider_stream_events` result.

## Compatibility

- The public `chat_with_collection_stream` arguments are unchanged.
- The final tool result payload remains unchanged.
- Raw MCP calls continue to use the JSON event result without FastMCP progress
  notifications.
- FastMCP clients without progress handlers still receive the final result.
