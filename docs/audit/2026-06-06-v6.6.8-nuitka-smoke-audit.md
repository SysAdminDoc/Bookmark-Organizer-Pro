# v6.6.8 Nuitka Smoke Audit

Date: 2026-06-06

## Findings

- The first smoke target imported `bookmark_organizer_pro.constants`, which
  loaded the package `__init__` and expanded the compile graph to 1,428 modules.
- That broad smoke compile exceeded the 10-minute bounded attempt and required
  stopping the leftover Nuitka/SCons/MSVC process tree.
- A self-contained smoke entrypoint avoids the full GUI dependency graph while
  still validating the local compiler, Windows metadata flags, icon handling,
  data-file inclusion, report emission, and executable startup.

## Changes Shipped

- Added `--target smoke` to `packaging/nuitka_build.py`.
- Added `packaging/nuitka_smoke.py`, a small console entrypoint that prints the
  app name and version.
- Preserved the default `--target app` behavior for the real GUI build path,
  including Tkinter plugin and package inclusion flags.
- Added tests for smoke target command generation and smoke metadata sync.
- Bumped release metadata to v6.6.8.

## Verification

- `py -3.12 packaging\nuitka_smoke.py --version` passed and printed
  `Bookmark Organizer Pro v6.6.8`.
- `py -3.12 packaging\nuitka_build.py --dry-run --target smoke --mode standalone --output-dir dist\nuitka-smoke --jobs 2` passed.
- `py -3.12 packaging\nuitka_build.py --target smoke --mode standalone --output-dir dist\nuitka-smoke --jobs 2` passed in about 18 seconds.
- Nuitka reported version 4.1.2, MSVC `cl 14.3`, 6 compiled C files, asset
  inclusion, icon post-processing, and `compilation-report.xml`.
- `dist\nuitka-smoke\nuitka_smoke.dist\BookmarkOrganizerProSmoke.exe --version`
  passed and printed `Bookmark Organizer Pro v6.6.8`.
- `py -3.12 -m compileall -q bookmark_organizer_pro tests packaging` passed.
- `py -3.12 -m pytest -q tests/test_packaging.py tests/test_browser_extension.py`
  passed with 11 tests.
- `py -3.12 -m pytest -q` passed with 309 tests.

## Remaining Distribution Hardening

- A full GUI Nuitka bundle should still be validated before replacing
  PyInstaller as the default release artifact.
- R-41 can begin from this smoke evidence by treating Nuitka as a candidate
  binary source while keeping PyInstaller available.
