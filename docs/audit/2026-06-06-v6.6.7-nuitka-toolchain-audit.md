# v6.6.7 Nuitka Toolchain Audit

Date: 2026-06-06

## Research Findings

- The declared `Nuitka>=4.1,<5.0` extra installed Nuitka 4.1.2 in the active Python 3.12 environment.
- `py -3.12 -m nuitka --version` reported MSVC `cl 14.3` as the available C compiler.
- Nuitka supports `--jobs=N` for limiting parallel C compiler jobs.

## Code Findings

- The first full-app standalone compile spawned many compiler processes and exceeded a 15-minute smoke timeout.
- The previous build helper had no way to throttle parallel compiler jobs.
- The aborted compile left ignored build output under `dist/` and required stopping the remaining compiler process tree.

## Changes Shipped

- Added `DEFAULT_JOBS = 4` to `packaging/nuitka_build.py`.
- Added `--jobs` command generation and CLI parsing.
- Added tests for default and custom job values.
- Bumped release metadata to v6.6.7.

## Verification

- `py -3.12 -m nuitka --version` passed and reported Nuitka 4.1.2 with MSVC `cl 14.3`.
- `py -3.12 packaging\nuitka_build.py --dry-run --jobs 2 --mode standalone --output-dir dist\nuitka-smoke` passed.
- `py -3.12 -m compileall -q bookmark_organizer_pro tests packaging` passed.
- `py -3.12 -m pytest -q tests/test_packaging.py tests/test_browser_extension.py` passed with 9 tests.
- `py -3.12 -m pytest -q` passed with 307 tests.
