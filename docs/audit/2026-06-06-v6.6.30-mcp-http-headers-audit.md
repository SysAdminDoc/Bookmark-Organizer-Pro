# v6.6.30 MCP HTTP Headers Audit

## Scope

R-15/R-58 transport hardening advanced after validating that standard FastMCP
URL clients do not send Bookmark Organizer Pro's mirrored `Mcp-Method` and
`Mcp-Name` headers. The middleware now treats those headers as optional while
still validating them when present.

## Changes

- `MCPHTTPHeaderValidationMiddleware` now allows requests without
  `Mcp-Method`.
- `MCPHTTPHeaderValidationMiddleware` now allows named requests without
  `Mcp-Name`.
- Mismatched provided `Mcp-Method` headers still return HTTP 400.
- Mismatched provided `Mcp-Name` headers still return HTTP 400.

## Compatibility

- Standard MCP clients that send only JSON-RPC bodies can reach the downstream
  Streamable HTTP app.
- Clients and proxies that provide mirrored headers still get validation.
- The JSON-RPC body replay path is unchanged.
