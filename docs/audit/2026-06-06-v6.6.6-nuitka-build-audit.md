# v6.6.6 Nuitka Build Audit

Date: 2026-06-06

## Research Findings

- Nuitka 4.1.1 is the current PyPI release as of 2026-06-06.
- Nuitka's official manual recommends `--mode=standalone` or `--mode=onefile` for executable builds.
- Nuitka's official manual documents `--windows-icon-from-ico` and `--windows-console-mode=disable` for Windows GUI apps.
- The current local Python environment did not have Nuitka installed before this cycle.

## Code Findings

- Existing PyInstaller packaging already centralizes app icon, version, product, and no-console metadata that the Nuitka build path should mirror.
- A dry-run command generator is the safest first build slice because it can be tested without requiring a local C compiler.
- Asset inclusion needs to keep the runtime `assets/` target path used by local development.

## Changes Shipped

- Added `packaging/nuitka_build.py`.
- Added onefile and standalone build modes.
- Added dry-run command printing.
- Added Tkinter plugin, package inclusion, app icon/data assets, Windows metadata, and compilation report flags.
- Added optional `nuitka = ["Nuitka>=4.1,<5.0"]` package extra.
- Bumped release metadata to v6.6.6.
