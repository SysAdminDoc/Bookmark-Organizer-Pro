# v6.6.24 Updater Staged Manifest Audit

## Scope

R-41 advanced from target cache staging to auditable staged-update readback.
Successful downloads now leave a manifest that can be inspected before any
future apply workflow exists.

## Changes

- Added `StagedUpdateStatus`.
- Added `UpdateManager.staged_manifest_path`.
- Added `UpdateManager.staged_update()` to read and validate
  `updates/staged_update.json`.
- Made successful `download_update()` calls write the staged manifest after all
  selected target paths pass the target-cache boundary guard.
- Added `updates staged` CLI output for absent, incomplete, or present staged
  updates.
- Documented the staged manifest in the updater bootstrap guide.

## Safety Boundaries

- `updates staged` only reads local manifest and file state.
- Manifest paths are validated to stay under `updates/targets/`.
- Missing staged files mark the staged update incomplete.
- No extraction is performed.
- No installer is launched.
- No application files are replaced.
- `updates apply` remains blocked.
