# v6.6.5 SQLite Runtime Audit

Date: 2026-06-06

## Research Findings

- The repo's storage operations already flow through `BookmarkManager`, making manager construction the right boundary for an opt-in backend selector.
- JSON must remain the default to preserve existing app and CLI behavior.
- Bookmark IDs are unsigned 64-bit values in current code, so SQLite cannot safely store them as signed INTEGER values.

## Code Findings

- `BookmarkManager` directly constructed `StorageManager`, so SQLite needed a small resolver rather than broad call-site changes.
- File watcher mtime checks can continue to use the resolved storage path.
- The v6.6.4 SQLite schema needed TEXT IDs to preserve generated bookmark IDs above SQLite's signed integer range.

## Changes Shipped

- Added `BookmarkManager(..., storage_backend="sqlite")`.
- Added automatic SQLite selection for `.sqlite`, `.sqlite3`, and `.db` file paths.
- Added `BOOKMARK_STORAGE_BACKEND=sqlite` for process-level opt-in.
- Kept JSON as the default and fallback backend.
- Migrated the SQLite schema to TEXT primary keys for bookmark IDs while preserving integer IDs in payload JSON.
- Bumped release metadata to v6.6.5.
