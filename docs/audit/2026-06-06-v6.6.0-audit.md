# v6.6.0 Autonomous Cycle Audit

Date: 2026-06-06

## Research Findings

- `tksheet` 7.6.0 is the current PyPI release, published 2026-03-11, and remains a fit for the R-16 list-virtualization item.
- The tksheet v7 documentation describes a pure-Python Tkinter table/treeview widget that draws only the visible portion of the table on canvases, which matches the large-library performance goal.
- The MCP draft caching spec now requires cache hints on list/discovery results, reinforcing R-58 as the next protocol-focused roadmap item.

## UX Findings

- The old list path rebuilt every visible bookmark row through `ttk.Treeview.delete()` and `insert()` on each refresh. That made filtering, sidebar refresh, and search scale with full row churn.
- Sidebar/chat bookmark deep links compared the first visible value column to the bookmark ID, so direct selection from those surfaces could silently fail.
- The tksheet replacement preserves the existing list workflow: multi-select, context menu, double-click open, header sort, row styling, zoom, and scroll behavior.

## Code Findings

- `BookmarkManager.batch()` only suppressed calls through `save_bookmarks()`. Direct mutation methods still called `storage.save()` inside batches, producing multiple writes.
- `NLQueryTranslator` had an internal heuristic parser but no public `heuristic_parse()` compatibility method.
- `DeadLinkScanner` migrated to typed records but lost the older `_results` and `_save_results()` compatibility surface.
- `SnapshotArchiver` exposed `snapshot()` but older callers expected `archive()`.
- Active release metadata had drifted across README, extension manifest, PyInstaller spec, and Windows version info after v6.5.2.

## Changes Shipped

- Added a tksheet-backed `VirtualBookmarkSheet` plus legacy `SortableTreeview` fallback.
- Switched the main bookmark list to bulk row replacement through the virtual table.
- Fixed bookmark-ID row selection for sidebar/chat deep links.
- Added `tksheet>=7.6,<8` to project dependencies and PyInstaller hidden imports.
- Bumped synchronized release metadata to v6.6.0.
- Added release metadata sync tests.
- Restored service compatibility methods and batch-save coalescing.
