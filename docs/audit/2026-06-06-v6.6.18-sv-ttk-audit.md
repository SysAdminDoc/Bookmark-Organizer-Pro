# v6.6.18 sv-ttk Theme Audit

Date: 2026-06-06
Scope: R-18, optional Sun Valley ttk theme integration

## Objective

Add an optional Windows 11-style ttk base theme without making `sv-ttk` a core
runtime dependency. Standard installs must continue to use the existing built-in
ttk fallback path and all BOP theme colors must remain authoritative.

## Changes

- Added optional `sv-ttk` loading in the desktop style manager.
- Added active-theme light/dark mode selection from `ThemeColors.bg_primary`.
- Applied the optional Sun Valley base before BOP's existing ttk color
  overrides.
- Preserved the previous `clam`/`default` fallback when `sv-ttk` is absent or
  cannot apply.
- Added `native_theme_status` diagnostics for the selected base theme and
  fallback error state.
- Added the `sunvalley` optional dependency group to package metadata.

## Compatibility

- Core installs do not require `sv-ttk`.
- Existing custom theme foreground, background, selection, and accent colors
  still flow through `StyleManager.apply_theme()`.
- If a theme is loaded before the Tk root exists, the selected colors are cached
  and applied after `StyleManager.initialize()`.
