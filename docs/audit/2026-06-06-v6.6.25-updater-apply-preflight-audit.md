# v6.6.25 Updater Apply Preflight Audit

## Scope

R-41 advanced from staged manifest readback to a non-mutating apply preflight.
The preflight reports staged update readiness and blockers without enabling
application.

## Changes

- Added `UpdateApplyPreflightResult`.
- Added `UpdateManager.apply_preflight()`.
- Added `updates apply --dry-run` and `updates apply --preflight`.
- Kept plain `updates apply` as a refusal gate.
- Documented the preflight command in the updater bootstrap guide.

## Safety Boundaries

- The preflight only reads staged update state.
- No extraction is performed.
- No installer is launched.
- No executable is run.
- No application files are replaced.
- The final blocker always keeps real update application disabled in this
  release.
