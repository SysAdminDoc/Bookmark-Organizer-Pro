# v6.6.27 Updater Apply Plan Audit

## Scope

R-41 advanced from cleanup to a non-mutating apply and rollback plan. The plan
describes the future apply sequence without creating directories, extracting
archives, replacing files, or enabling application.

## Changes

- Added `UpdateApplyPlan`.
- Added `UpdateManager.build_apply_plan()`.
- Added `updates plan`.
- Added a deterministic rollback path based on current and target versions.
- Documented the plan command in the updater bootstrap guide.

## Safety Boundaries

- The plan only reads staged update state.
- No rollback directory is created.
- No extraction is performed.
- No installer is launched.
- No executable is run.
- No application files are replaced.
- `updates apply` remains blocked.
