# v6.6.14 MCP Chat Response Event Audit

Date: 2026-06-06
Scope: R-15 partial, MCP streaming for RAG chat

## Objective

Add a client-friendly streaming surface for MCP RAG chat without overstating
the current provider capabilities. The existing provider abstraction returns a
completed answer, so this release adds deterministic response events around
that answer and leaves provider-native token streaming open.

## Changes

- Added `ChatStreamEvent`, bounded chunk sizing, answer chunk splitting, and
  event construction helpers in the RAG chat service.
- Added `CollectionChat.stream_answer()` as a shared service entry point for
  chunked response events.
- Added MCP tool `chat_with_collection_stream` with the same question,
  `restrict_ids`, `restrict_tag`, and `restrict_category` inputs as
  `chat_with_collection`, plus bounded `chunk_chars`.
- Shared bookmark ID/tag/category scoping between the existing chat tool and
  the new stream-shaped tool.
- Registered the new tool in both raw MCP and FastMCP catalogs with read-only
  and open-world annotations.
- Mirrored existing MCP chat auth scope behavior by requiring read-write tokens
  when token auth is configured.

## Result Shape

`chat_with_collection_stream` returns:

- `mode`: `chunked_response_events`
- `provider_streaming`: `False`
- `events`: ordered `chunk` events followed by one `complete` event containing
  sources and chunk provenance
- `answer`: the full completed answer for clients that need a fallback

## Remaining R-15 Work

- Add provider-native token streaming once the app's provider client interface
  exposes streaming completions.
- Map provider tokens to MCP progress or transport-level streaming events once
  the FastMCP integration path is selected and tested.
- Preserve the v6.6.14 event shape as the compatibility fallback for clients
  that do not consume live progress notifications.
