# v6.6.4 SQLite Foundation Audit

Date: 2026-06-06

## Research Findings

- SQLite WAL mode allows concurrent readers while a writer commits, which is the concurrency model needed before a future web client can share the bookmark library.
- WAL is best suited to local same-host databases, matching BOP's local-first data model.
- Python's stdlib `sqlite3` is sufficient for the initial storage backend; no new runtime dependency is required.

## Code Findings

- JSON persistence is already centralized behind `StorageManager`, so an opt-in SQLite backend can mirror the existing `load()`/`save()` shape.
- Bookmark data has many evolving fields, so the safest first SQLite schema stores the full bookmark payload JSON and indexes only high-value query columns.
- JSON should remain the default until the app has an explicit runtime backend selection path.
- On Windows, sqlite3 connection context managers do not close file handles, so the backend must explicitly close every connection.
